# Contributing

Keep changes narrow, reproducible, and evidence-first.

1. Create a focused branch.
2. Add a fixture or test that fails before the change when practical.
3. Implement the smallest deterministic fix.
4. Run `make test` and `make demo`.
5. Explain remaining uncertainty in the pull request.

Do not submit generated simulator results without the commands, versions, configuration, and raw artifacts needed to reproduce them.
