# v0.8.1 — Keyless Signed Evidence Release

`v0.8.1` hardens release provenance without claiming a new hosted inference experiment.

## What this release proves

The deterministic Cerebras evidence ZIP, its SHA-256 checksum, and its release provenance sidecar are signed by the repository's GitHub Actions release workflow using a short-lived OIDC-backed Sigstore certificate.

The workflow then downloads every published asset and verifies:

- exact byte equality with the locally built files;
- SHA-256 and size bindings;
- repository, tag, commit, and workflow provenance;
- GitHub Artifact Attestation identity through the online service;
- the same signature through the published Sigstore bundle;
- rejection of attestations produced on self-hosted runners.

## Evidence payload

This patch release intentionally reuses the exact verified Cerebras milestone preserved by `v0.8.0`:

- model `gpt-oss-120b`;
- HTTP `200`;
- evidence status `COMPLETE`;
- TTFT `203.354406 ms`;
- provider-timed output throughput `1912.297310144391 tokens/s`;
- SDK `1.67.0`;
- manifest verification `3/3` with zero mismatches.

The canonical source remains `docs/evidence/releases/v0.8.0/cerebras-live/`. The explicit pointer in `docs/releases/v0.8.1.evidence-source` prevents duplicating or silently changing those evidence files.

## Release assets

A successful release publishes:

```text
v0.8.1-cerebras-live-evidence.zip
v0.8.1-cerebras-live-evidence.zip.sha256
v0.8.1-cerebras-live-evidence.zip.provenance.json
v0.8.1-release-attestation.sigstore.json
```

## Trust boundary

The attestation proves the origin and integrity of the exact published files. It does not independently prove provider hardware behavior, model quality, silicon correctness, energy efficiency, formal verification, coverage closure, or vendor-wide performance.
