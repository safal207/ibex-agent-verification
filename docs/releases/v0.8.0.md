# v0.8.0 — Dual-Rail Evidence Milestone

`v0.8.0` adds a provider-neutral hosted inference evidence rail alongside the existing Ibex silicon evidence rail.

## Highlights

### Real hosted Cerebras evidence

The release preserves the first fully green live run:

- model `gpt-oss-120b`;
- HTTP `200`;
- evidence status `COMPLETE`;
- TTFT `203.354406 ms`;
- provider-timed output throughput `1912.297310144391 tokens/s`;
- SDK `1.67.0`;
- independently verified manifest with `3/3` files and zero mismatches.

The sanitized request, timestamped stream capture, analysis, manifest, receipt, and verification report are committed under `docs/evidence/releases/v0.8.0/cerebras-live/` and attached to the GitHub Release as a deterministic ZIP asset.

### Correct reasoning-model throughput

The analyzer now prefers provider-reported completion time when provider usage and reasoning-token details are present. It refuses to publish a misleading client-derived rate when reasoning tokens exist but provider completion timing is absent.

### Secure live runner

The Cerebras runner reads credentials only from the environment, fixes the endpoint, disables retries and TCP warming, rejects secret-like request fields recursively, records only safe response headers, and preserves verified failure evidence for real request failures.

### Corrected hosted Ibex counts

The current hosted causal record is `385 MEMORY_WAIT`, `248 INSTRUCTION_FETCH_WAIT`, `6 INTERRUPT_SERVICE`, and `241 UNKNOWN`.

## Release asset

The release workflow verifies the committed evidence manifest, builds a deterministic ZIP, uploads it to the GitHub Release, and verifies the published asset metadata.

## Trust boundary

This release does not prove internal provider hardware, RTL correctness, model quality, energy efficiency, vendor-wide performance, formal silicon correctness, coverage closure, or tape-out readiness. It preserves auditable evidence for specific pinned experiments.
